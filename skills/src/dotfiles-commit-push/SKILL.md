## 目的

コミット・push を安全に行う（秘密情報を混入させない）。

## チェック

- `git status` / `git diff` で意図した変更のみか確認
- 機密情報（鍵・トークン・認証情報・個人情報）が含まれていないか確認
- コミットメッセージは必ず規約フォーマットに従う（「なぜ」を中心に）

## コミットメッセージ規約（必須）

コミットメッセージは必ず以下の 4 要素で構成する。

- Emoji（コミットの種類）
- Title（コミットの概要）
- Reason（コミットの理由）
- Specification（コミットの意図・仕様）

テンプレ:

```text
<Emoji> <Title>

Reason:
- ...

Specification:
- ...
```

## 方針

- 破壊的操作（force push 等）は原則しない。必要な場合は理由と影響を明記する。
