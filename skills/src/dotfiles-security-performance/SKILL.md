## 目的

設計・実装・レビューで **セキュリティとパフォーマンスを最優先**に判断する。

## 必須チェック（セキュリティ）

- **入力検証**: 外部入力は全て検証（型・長さ・範囲・形式）
- **認証/認可**: エンドポイント/ユースケース単位で漏れがないか
- **秘密情報**: トークン/キー/個人情報をログ・エラー・レスポンス・コミットに出さない
- **安全なデフォルト**: deny by default / least privilege
- **依存関係**: 新規追加は必要最小限。公式ドキュメントと脆弱性を確認

## 必須チェック（パフォーマンス）

- **I/O**: ループ内の DB/API 呼び出し、N+1 を避ける
- **メモリ**: 余計なコピーやアロケーションを避ける
- **バッチ/ページング**: 大量データは一括読み込みしない
- **計測**: 重要パスは計測/可観測性（ログ/メトリクス/トレース）を意識

## 出力フォーマット

- 重要度を Must / Should / Nice to have に分けて指摘する
